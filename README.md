# Evolution: xenonpass is now an Organization

To everyone,

`xenonpass` has outgrown the "single repository" phase. 

What started as a personal project to secure my own digital footprint has evolved into a complex architecture that demands more than just a folder full of code. With a C-based vault, a Rust-powered TUI, and more components to come, we aren't just building a password manager—we are building a system.

Keeping everything in a single repo was fine for an MVP. It is a bottleneck for what we’re doing now.

To ensure proper modularity, cleaner CI/CD pipelines, and a structure that actually scales, `xenonpass` is officially transitioning into an organization. This isn't just a rename; it’s a commitment to treating this project with the engineering rigor it deserves. 

We’re here to prove that security doesn't have to be bloated, and performance doesn't have to be compromised. If you’re here for the "bare-metal" philosophy, you’re in the right place. 

The architecture is set. The roadmap is live. Let’s build something that lasts.

-- Mohammed Sajid Shaik
